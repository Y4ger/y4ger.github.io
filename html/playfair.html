Playfair Cipher

What is it?

A playfair cipher is an encryption technique invented in 1854 by Charles Wheatstone. It is virtually impossible to break without the keyword.

This cipher is created in four steps:

  1. Choose a keyword
  2. Adapt the keyword
  3. Create a keytable
  4. Encrypt your message

Keyword

The first step when creating a Playfair is to choose a keyword. This is the easy part. Any work or phrase will suffice.

Adapt

There are some rules that need to be applied to your keyword before it is transformed into a keytable.

First, all J's must be replaced with I's. This must be done to shrink the alphabet from 26 letters to 25 letters. The number of letters will be made obvious during the key table creation.

Second, all spaces must be removed. This rule only applies if your keyword is a phrase.

Third, all letters can only occur once. That means if your keyword is "hello" it should be adapted to "helo"

Keytable

Now that the keyword has been adapted you can create the keytable. A keytable is 5 by 5, meaning there are 25 available cells (that is why we had to shrink the alphabet).

Start the keytable by placing the first letter in your keyword in the top-left cell. Then place the second letter in the next cell of that row. Continue this until every letter in your keyword has been placed.

Then fill the remaining cells with the letters of the alphabet that haven't been placed yet, in alphabetical order. Don't forget to skip J!

Encrypt

Now that you have your keytable, its time to start encrypting. Choose the message you want to make secret.

The encryption process is easy once you understand the process.

First, apply the same rules that were used when adapting the keyword to the message.

Second, separate all of the letters into pairs of two starting with the first two letters. If any of the pairs end up being the same letter twice ("LL") replace the second letter with an "X" ("LX").

Now find the location of the first pair of letters on your keytable. There are three possible arrangements for any letter pairs.

  1. Same Row
  2. Same Column
  3. Different Row and Column

If the two letters are in the same row they become the letter directly to their right (unless they are already on the far right then they become the first letter of that row)

If they are in the same column the letters become the letter directly below them (unless the letter is already at the bottom then it becomes the first letter in that column)

If the letters are not in the same row or column the encryption is a little more tricky. Each letter becomes the letter that is in its own row but shares a column with the other letter of the pair.

It is important that when you encrypt your message you keep the letter order how it was originally
